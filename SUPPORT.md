# Support

This repository is maintained as a public automation template and portfolio project.

## Getting Help

Before opening an issue:

1. read the README;
2. check the setup checklist;
3. review the architecture documentation;
4. test with demo placeholders only;
5. confirm that no private lead data is included.

## What to Include in an Issue

- short problem description;
- steps to reproduce;
- expected behavior;
- actual behavior;
- whether the issue is related to the website widget, Telegram bot, n8n, PostgreSQL, or notifications;
- safe logs with secrets removed.

## What Not to Include

Do not include Telegram tokens, PostgreSQL passwords, n8n credentials, private webhook URLs, real chat IDs, real leads, customer phones, customer emails, or `.env` files.

## Production Note

A production deployment should include access control, consent tracking, backups, monitoring, and a clear privacy policy.

## Project Status

This is not a commercial support channel. Issues and improvements are handled as time allows.
