# Public Demo Notes

Use this note when preparing a public demo of the repository.

## Demo Goals

- Show the website-to-Telegram flow clearly.
- Explain the system architecture.
- Keep setup steps understandable.
- Use demo values only.
- Keep privacy notes visible.

## Before Sharing

- Check that the README is current.
- Check that examples are generic.
- Check that screenshots do not reveal real leads.
- Check that setup notes match the current workflow.
